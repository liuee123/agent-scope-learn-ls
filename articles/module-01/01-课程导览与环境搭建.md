## 核心内容

### 1. 课程全景：五个模块学完能做什么

在开始写代码之前，先花 5 分钟了解这门课的整体结构。

**模块一：入门篇（第 1-7 节）** —— 从零认识 AgentScope。你会搭建环境、创建第一个 Agent、理解 ReActAgent 和 HarnessAgent 的双层架构、掌握对话管理、上下文压缩、Workspace 工作区等基础设施。

**模块二：进阶篇（第 8-16 节）** —— 掌握 Agent 的核心能力。Tool 体系让 Agent 能查订单、查天气；Middleware 中间件体系让你能在 Agent 生命周期的每个环节插入自定义逻辑；RAG 知识库让 Agent 能检索企业内部文档；MCP 协议让你能连接外部 Agent 生态。

**模块三：高级篇（第 17-24 节）** —— 多 Agent 协作。Subagent、Supervisor、Handoffs、Pipeline、A2A 五种模式，从单 Agent 走向多 Agent 编排。Plan Mode 让 Agent 先规划再执行，沙箱隔离让子 Agent 安全运行。

**模块四：实战篇（第 25-30 节）** —— 贯穿全课程的大项目：一个完整的「多 Agent 智能客服系统」。六个 Agent 角色协作（客服主管、意图识别、售前顾问、售后专员、技术支持、升级处理），从架构设计到部署上线全流程。

**模块五：源码分析篇（第 31-41 节）** —— 11 节课深入 AgentScope 源码。ReActAgent 推理循环、Middleware 洋葱模型、Tool 调用链路、Workspace 沙箱抽象……学会阅读源码，不再被框架"黑盒"困住。

学完 41 节课，你将能够：独立设计多 Agent 系统架构、用 AgentScope 实现企业级 AI 应用、阅读并理解 Agent 框架的底层源码。

### 2. JDK 21 安装与验证

AgentScope Java 版最低要求 JDK 17，本课程使用 **JDK 21**（当前最新的 LTS 版本）。

**Windows 用户：**

或者使用 [Oracle JDK 21](https://www.oracle.com/java/technologies/downloads/#java21)，注意选择 **x64 Installer** 或 **ARM64 Installer**（M 系列 Mac）。

**macOS 用户：**

```bash
# 方式一：Homebrew
brew install openjdk@21

# 方式二：SDKMAN（推荐，方便后续切换版本）
curl -s "https://get.sdkman.io" | bash
sdk install java 21.0.6-librca
```

**验证安装：**

打开终端（Windows 用 PowerShell 或 Git Bash），输入：

```bash
java -version
```

预期输出类似：

```
openjdk version "21.0.6" 2025-01-21 LTS
OpenJDK Runtime Environment (build 21.0.6+10-LTS)
OpenJDK 64-Bit Server VM (build 21.0.6+10-LTS, mixed mode, sharing)
```

> ⚠️ **常见坑**：如果 `java -version` 显示 Java 8 或 Java 11，说明环境变量没有指向 JDK 21。需要检查 `JAVA_HOME` 和 `PATH` 的配置顺序。

### 3. Maven Wrapper：零配置的构建工具

本课程每个代码目录都自带 **Maven Wrapper**（`mvnw` 脚本）。你不需要在电脑上单独安装 Maven——只要 JDK 21 装好了，执行 `mvnw` 就会自动下载正确的 Maven 版本（3.9.9）并构建项目。

> 💡 **什么是 Maven Wrapper？** 它是 Maven 官方提供的脚本机制。项目的 `.mvn/wrapper/` 目录里记录了「用哪个版本的 Maven」，执行 `mvnw` 时会自动下载并使用该版本。这样做的好处是：全班所有同学使用完全相同的 Maven 版本，彻底消除「你的能跑我的不能跑」的环境差异问题。

验证方式（后续项目创建好后执行）：

```bash
./mvnw --version   # macOS / Linux / Git Bash
mvnw.cmd --version # Windows CMD / PowerShell
```

### 4. 创建我们的第一个项目

我们手动创建项目结构，不使用 Spring Initializr。这样你将对项目的每一行配置都了如指掌。

#### 4.1 目录结构

```
section-01/
├── pom.xml                        # Maven 配置（核心）
├── mvnw                           # Maven Wrapper 脚本（Unix）
├── mvnw.cmd                       # Maven Wrapper 脚本（Windows）
├── .mvn/
│   └── wrapper/
│       └── maven-wrapper.properties  # Maven 版本配置
├── .gitignore
└── src/
    └── main/
        ├── java/
        │   └── io/agentscope/course/
        │       └── Application.java     # Spring Boot 启动入口
        └── resources/
            └── application.yml          # Spring Boot 配置
```

#### 4.2 手写 pom.xml

在项目根目录创建 `pom.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>4.0.4</version>
        <relativePath/>
    </parent>

    <groupId>io.agentscope.course</groupId>
    <artifactId>section-01-hello-agentscope</artifactId>
    <version>1.0.0</version>
    <name>Section 01 — Hello AgentScope</name>

    <properties>
        <java.version>21</java.version>
        <agentscope.version>2.0.1-SNAPSHOT</agentscope.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter</artifactId>
        </dependency>

        <dependency>
            <groupId>io.agentscope</groupId>
            <artifactId>agentscope-harness</artifactId>
            <version>${agentscope.version}</version>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
```

逐行解读：

- **`<parent>`**：继承 Spring Boot 4.0.4 的父 POM，它管理了上百个依赖的版本号，你不用再一个个指定。Spring Boot 4.x 是 2025 年末发布的新一代版本，底层基于 Spring Framework 7.x，需要 JDK 17+。
- **`<java.version>21</java.version>`**：告诉 Maven 编译器使用 Java 21 编译。Spring Boot 父 POM 会自动读取这个属性来设置 `maven.compiler.source` 和 `maven.compiler.target`。
- **`<agentscope.version>2.0.1-SNAPSHOT</agentscope.version>`**：定义一个属性变量，方便在多处引用相同的版本号。
- **`spring-boot-starter`**：Spring Boot 核心启动器，包含自动配置、日志（Logback）、YAML 支持等基础能力。第 1 节暂时不需要 Web 能力（`spring-boot-starter-web`），所以只引入最精简的 starter。
- **`agentscope-harness`**：AgentScope 的工程化包装层，**传递引入** `agentscope-core`（推理核心），所以你只需声明这一个依赖就够了。

#### 4.3 安装 AgentScope 到本地仓库

AgentScope 2.0.1-SNAPSHOT 是快照版本（还未发布到 Maven Central），你需要先将源码编译安装到本地 Maven 仓库（`~/.m2`）。

```bash
# 克隆源码
git clone https://github.com/agentscope-ai/agentscope-java.git
cd agentscope-java

# 安装到本地（跳过测试，加快速度）
mvn install -DskipTests
```

看到 `BUILD SUCCESS` 后，你的本地 `~/.m2/repository/io/agentscope/` 目录下就有了 `agentscope-core` 和 `agentscope-harness` 的 jar 包。这一步只需执行一次，后续所有课程项目都能直接引用。

> 💡 **为什么要 `mvn install`？** Maven 的依赖解析顺序是：本地仓库（`~/.m2`）→ 远程仓库（Maven Central / 阿里云镜像）。快照版本通常不会发布到 Central，所以需要手动安装到本地。等 AgentScope 2.0.1 正式发布后，这一步就可以省略了。

#### 4.4 编写启动类

创建 `src/main/java/io/agentscope/course/Application.java`：

```java
package io.agentscope.course;

import io.agentscope.core.Version;
import org.springframework.boot.CommandLineRunner;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application implements CommandLineRunner {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }

    @Override
    public void run(String... args) throws Exception {
        System.out.println("========================================");
        System.out.println("  AgentScope 课程项目启动成功！");
        System.out.println("========================================");
        System.out.println("  AgentScope 版本 : " + Version.VERSION);
        System.out.println("  JDK 版本        : " + System.getProperty("java.version"));
        System.out.println("  操作系统         : " + System.getProperty("os.name"));
        System.out.println("  User-Agent      : " + Version.getUserAgent());
        System.out.println("========================================");
    }
}
```

这里用到了三个关键点：

1. **`@SpringBootApplication`**：Spring Boot 的万能注解，等价于 `@Configuration` + `@EnableAutoConfiguration` + `@ComponentScan` 三个注解的合体
2. **`CommandLineRunner`**：Spring Boot 启动完成后的回调接口。`run()` 方法会在 `SpringApplication.run()` 完成后立即执行
3. **`Version.VERSION`**：AgentScope 内置的版本常量，来自 `io.agentscope.core.Version` 类。用于验证依赖确实在 classpath 上

#### 4.5 编写 application.yml

创建 `src/main/resources/application.yml`：

```yaml
server:
  port: 8080

spring:
  main:
    banner-mode: off       # 关闭 Spring Boot 启动 banner（让输出更干净）
  application:
    name: section-01-hello-agentscope
```

暂时不配置 API Key——那是第 2 节的内容。第 1 节只需要证明「项目能编译、能启动、AgentScope 在 classpath 上」这三点。

### 5. IntelliJ IDEA 配置

本课程全程使用 IntelliJ IDEA（Community 版免费可用）。

**打开项目：**

1. 启动 IDEA → File → Open → 选择 `section-01/` 目录 → OK
2. IDEA 会自动识别为 Maven 项目，弹出「Maven build scripts found」→ 点击 **Load Maven Build Scripts**
3. 等待右下角进度条走完（首次打开需要下载依赖，可能需要 2-5 分钟）

**关键设置：**

- **Project Structure → Project SDK**：确认为 JDK 21，若不是则手动添加
- **Settings → Build → Compiler → Java Compiler**：确认 `Target bytecode version` 为 21
- **Settings → Maven**：确认 `Maven home path` 使用的是 Bundled（IDEA 自带的 Maven），但**实际构建时我们仍然用命令行 `mvnw`**，这样可以保证构建结果与 IDEA 无关

> ⚠️ **常见坑**：IDEA 有时会默认使用自带的 JDK（如 JetBrains Runtime 17），需要在 Project Structure 中手动切换。如果 IDEA 内 Maven 面板的依赖树一直转圈，试着点一下右上角的「Reload All Maven Projects」按钮（圆形箭头图标）。

### 6. DeepSeek 与 DashScope API Key 申请

本课程使用 **双模型** 策略：DeepSeek 作为主力推理模型，DashScope（阿里云百炼）作为降级备选模型。第 2 节开始会真正调用模型，但 API Key 现在就可以申请好。

#### DeepSeek API Key

1. 打开 [https://platform.deepseek.com](https://platform.deepseek.com)
2. 注册/登录 → 左侧菜单「API Keys」→ 点击「创建 API Key」
3. 复制生成的 Key（只显示一次！），保存到安全的地方
4. DeepSeek 新用户通常赠送一定额度，足够跟完本课程

#### DashScope API Key

1. 打开 [https://dashscope.aliyun.com](https://dashscope.aliyun.com)
2. 使用阿里云账号登录 → 控制台 →「API-KEY 管理」→ 创建 Key
3. 同样保存好 Key

**安全提醒：不要把 API Key 硬编码在代码里或提交到 Git！** 我们将在第 2 节通过环境变量或 `application.yml` 的外部化配置来管理。可以先把 Key 写到 `~/.bashrc` 或 `~/.zshrc` 中（Windows 用户写到系统环境变量）：

```bash
export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxx"
export DASHSCOPE_API_KEY="sk-xxxxxxxxxxxxxxxx"
```
